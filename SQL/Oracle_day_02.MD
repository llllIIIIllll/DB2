Oracle 体系
+物理结构
|-----+数据文件
      |------+ .dbf格式(数据文件用来存储数据库中的全部数据，如数据库表中的数据和索引数据)
      +日志文件
      |------+ Log*.dbf格式(日志文件(又称重做日志文件),记录数据库所做的全部变更，故障时恢复)
      |------+ 在事务信息将被覆盖时，是否应该将文件归档，数据库分归档模式：ARCHIVELOG(归档日志)或NOARCHIVELOG(非归档日志)模式。
      +控制文件
      |------+ Ctr*.ctl格式(用于打开、存取数据库。它们是较小的二进制文件，其中记录了数据库的物理结构。)
      |------+ 每个数据库可以有一个或多个，控制文件只对应一个数据库
      +配置文件
      |------+ initsid*.ora格式(ASCII文本文件，记录Oracle数据库运行时的一些重要参数)

+逻辑结构
|-----+表空间
      |------+ 分类
             |------+ 数据表空间(存放数据对象)
             |------+ System表空间
             |------+ 回滚表空间(Rollback TableSpace)
             |------+ 临时表空间(Temp TableSpace)
      |------+ 使用
             |------+ 数据字典与用户数据分离
             |------+ 回退数据与用户数据分离
             |------+ 数据文件分散到不同硬盘，平均分布物理I/O
             |------+ 将某个表空间设置为脱机状态或联机状态(对数据库某部分备份和恢复)
             |------+ 某个表空间设为只读(数据库的一部分只读)
             |------+ 特殊用途专设一个表空间(如临时表空间)优化使用效率
             |------+ 灵活配置空间余额
      |------+ 对象的存储
             |------+ 同一对象可存于表空间中的任意一个或多个数据文件中
|-----+段
      |------+ 数据段 (表中的记录)
      |------+ 索引段 (在Oracle数据库中每个未分区索引都有一个索引段保存索引中的索引条目。对于一个分区索引，每个分区都有一个索引段保存它的数据。)
      |------+ 临时段 (当处理查询时，Oracle可能会需要使用到一些临时存储空间，用于临时保存解析过的查询语句以及在排序过程中产生的临时数据。Oracle会自动在专门用于存储临时数据的表空间为操作分配临时段)
      |------+ 回退段 (回滚段用于存放数据修改之前的值(数据修改之前的位置和值))
             |------+ 事务回滚
             |------+ 事务恢复
             |------+ 读一致性
|-----+区
      |------+ 连续的数据块组成的数据库存储空间(缺省下，数据库用表空间中存储的存储参数来管理)
             |------+ INITIAL      : 段建立时分配的第一个区大小(字节)
             |------+ NEXT         : 段分配下一个增量区总数(字节)
             |------+ MAXEXTENTS   : 分配给段的区总数
             |------+ MINEXTENTS   ：建立段时分配的区总数，段最小区数量
             |------+ PCTINCREASE  : 为段分配的每个增量区超过上一个区的百分比
             |------+ INITRANS     ：为初始数量的DML事务条目(并发访问数据块中的行)保留的预分配空间数量
             |------+ NTKXTRANS    : 当多个事务同时访问同一数据块的行时，为数据块中每个DML事务的条目分配的空间。
|-----+数据库块
      |------+逻辑块或Oracle块(对应磁盘上一个或多个物理块)(初始化参数db_block_size(init.ora)决定大小,典型是2k)
             |------+ Pckfree 和 Pctused 优化数据块空间 使用。
                    |------+ PCTFREE ：块中保留用于UPDATE操作的空间百分比，当数据占用的空间达到此上限时，新的数据将不能再插入到此块中；
                    |------+ PCTUSED ：指定块中数据使用空间的最低百分比；
|-----+模式对象
      |------+ 表、视图、序列、同义词、索引、触发器、存储过程等。

+内存结构
|-----+系统全局区
      |------+ SGA，System Global Area 为一个实例分配的一组共享内存缓冲区，保存着Oracle系统与所有数据库用户的共享信息，包括数据维护、SQL语句分析，重做日志管理等。是实例的主要部分。
             |------+ 数据块缓冲区 (用户数据高速缓冲区)
                    |------+ 最近从数据文件中读取的数据块 为所有与该实例相链接的用户进程所共享。容量受物理容量限制(DB_ACHESIZE)
             |------+ 字典缓冲区 保存数据字典中的行
                    |------+ 最近最少使用(LRU)算法管理 是SQL共享池的一部分，共享池的大小(字节)数据库文件init.ora中的SHARED_POOL_SIZE
             |------+ 重做日志缓冲区
                    |------+ 对数据库进行修改的任何事务(Transaction)在记录到重做日志之前都必先放到重做日志缓冲区(Redo Log Buffer)中
                    |------+ 重做日志缓冲区是专为此开辟的一块内存区域，重做日志缓存中的内容将被LGWR后台进程随时写入重做日志文件。
                    |------+ 重做日志缓存是一个循环缓存区，在使用时从顶端向底端写入数据，然后再返回到缓冲区的起始点循环写入。
                    |------+ 重做日志缓冲区的大小(字节)init.ora的LOGBUFFER
             |------+ 共享SQL池 (Shared SQL Pool)  相当于程序高速缓冲区，所有的用户程序都存放在共享SQL池中。
                    |------+ SQL共享池包括库高速缓存、数据字典高速缓存和服务器控制结构。
                    |------+ init.ora的SHARED_POOL_SIZE(字节)。用户必须将这个值设得足够大，以确保有足够的可用空间来装载和存储PL/SQL块和SQL语句。
             |------+ Java池
                    |------+ Java池为Java命令提供语法分析。
             |------+ 多缓冲池
                    |------+ 大数据集与其他的应用程序分开，以减少它们争夺数据块缓冲区内相同的资源。可以在SGA中创建多个缓冲池 。
|-----+程序全局区
      |------+ Program Global Area，PGA 是单个Oracle进程使用的内存区域，不属于实例的内存结构。它含有单个进程工作时需要的数据和控制信息，非共享，只有服务进程本身才能够访问它自己的PGA区。
|-----+排序区
      |------+ 排序区存在于请求排序的用户进程的内存中，由于排序需要内存空间，Oracle利用该内存排序数据，这部分空间称为排序区。该空间的大小为适应排序数据量的大小，可增长，但受初始化参数SORT_AREA_SIZER限制
|-----+软件代码区
      |------+ Software Code Area
             |------+ 存储正在执行的或可以执行的程序代码。只读，可安装成共享或非共享。Oracle系统程序是共享的，多个Oracle用户可存取它，而不需要在内存有多个副本。用户程序可以共享也可以不共享。

+数据库实例与进程
|-----+Oracle数据库实例
      |------+ 数据库是指物理上的数据库文件或逻辑上的数据库结构 基于之上管理和控制物理数据库的软件系统，称为数据库管理系统(DBMS)。
      |------+ 数据库实例是指软件系统中用来访问数据库文件集的存储结构以及后台进程的集合，它是存取和控制数据库的软件机制。  

|-----+进程
      |------+ 进程与程序的区别在于前者是一个动态概念，后者是一个静态实体；程序仅仅是指令的有序集合，而进程则强调执行过程。进程可以动态地创建，完成任务后即会消亡。
      |------+ 进程又称任务 一个进程执行一组操作 完成一个特定的任务。对Oracle数据库管理系统来说，用户进程  服务器进程  后台进程
             |------+ 用户进程
                    |------+ 用户运行一个应用程序时，系统就为它建立一个用户进程。用户进程执行的是一个应用程序或Oracle工具程序的代码，以完成用户所指定的任务。用户进程不是实例。
             |------+ 服务器进程
                    |------+ 服务器进程处理与应用程序相连的用户进程的请求，它与用户进程相通讯，为相连的用户进程的Oracle请求服务。
             |------+ 后台进程
                    |------+ SMON : 系统监控进程 (system monitor)
                    |------+ PMON : 进程监控进程 (process monitor)
                    |------+ DBWR : 数据库写进程
                    |------+ LGWR : 日志文件写进程
                    |------+ ARCH : 归档进程(archiver process)
                    |------+ RECO : 恢复进程
                    |------+ LCKN : 封锁进程

  |------+ 为框架对象如何分配和使用空间

  |------+ 完整性约束信息
  |------+ 数据库安全信息，包括用户、权限、角色等
  |------+ 审计信息
|-----+数据字典的结构
      |------+ 基础表
             |------+ 存储相关数据库信息的表，这些数据库的信息包括表、索引、约束，以及所有其他数据库对象结构的信息。
             |------+ 属于SYS用户，通过运行SQL脚本来创建(在数据库创建时自动发生)
             |------+ 是Oracle系统的核心。只有Oracle才能写和读取这些表。
             |------+ 基础表中的存储的信息通常是经过加密处理的。
             |------+ 大部分数据字典基础表的名称中都包含"$"等特殊字符。
      |------+ 动态性能视图
             |------+ 在操作过程中，Oracle维护了一种“虚拟”表的集合，记录当前数据库的活动。
             |------+ SYS拥有动态性能表，其名字都是以v_$开头的。
             |------+ 在这些表上面创建的视图被称为动态性能视图(dynamic performance view)。
             |------+ 可为这些视图创建公共同义词，同义词名称以V$开头。
      |------+ 数据字典视图
             |------+ ALL视图
                    |------+ 包含了用户查询表时可以访问的所有对象的信息。
             |------+ USER视图
                    |------+ 前缀为USER 每个数据库用户都拥有一套属于自己的USER视图。在USER视图中包含了该用户模式下所有对象的信息
             |------+ DBA视图
                    |------+ 前缀为DBA，在DBA数据字典视图中包含着全部数据库对象的信息。
                    |------+ 对于带有DBA前缀的视图，显示了整个数据库的情况。因此，它们只能被数据库管理员查询。
                    |------+ 授予系统权限SELECT ANY TABLE的用户都能查询带有DBA前缀的视图。
|-----+数据字典的用途
      |------+ 对于Oracle系统本身而言，当数据库实例运行时，会需要使用数据字典基础表中的信息。
      |------+ Oracle从基础表中读取信息，来判断用户要求访问的对象是否存在。
      |------+ 同时，当用户对数据库结构、对象结构做出修改时，Oracle向基础表中写入相应的修改信息。
